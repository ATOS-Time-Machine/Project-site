Home Requirements

Initial Requirements

  • Redesign of current GUI to be more user friendly (making use of current logos and themes)
  • User Logon feature which either makes use of the current network logon credentials or has a password reset functionality in order allows users to access the website
  • Ability to add permissions to certain users to limit sections of the website that they can access
  • Functionality to allow users with the correct permissions to be able to set up groups showing a department's structure with the ability to assign roles i.e. engineer/team manager/reporter/administrator
  • Functionality to generate overtime reports based on the information selected by users using simple filters, and the ability to send these monthly to users on request if they have the correct permissions
  • Functionality to generate the special claims form (again with the ability to send these monthly to users with appropriate permissions) in the necessary format
  • Functionality to be able to integrate with the current ESS system used to determine employee pay

Time machine shall have a tab which shows all of the overtime requests that have been approved and confirmed by the user. Functional Standard User Should

Time machine will be able to allocate certain users with the correct permissions as an approved for a selection of other users. Functional Team Leaders Must

Use Case Logging in
Description A user attempts to log in to the system
Primary actor Standard users
Secondary actor None
Precondition None
Main Flow
  1. The user loads up the webpage
  2. The user enters their email in the appropriate field
  3. The user must enter their password in the appropriate field
  4. The user must either use the 'log on' button or press enter
Post-Condition The user should land on the homepage
Use Case Overtime Request
Description A user attempts to submit an overtime request
Primary actor Standard users
Secondary actor None
Precondition The user must have logged into the system
Main Flow
  1. The user must head to the overtime request tab
  2. They must select the option to create a new request
  3. They must then fill in all of the field
  4. They must then submit the overtime request
Post-Condtion The overtime request will then go to their team leader for approval, and then request the user confirm that they did follow what their request claims
Use Case Overtime Report
Description A team leader attempts to generate an overtime report
Primary actor Team leaders
Secondary actor Standard users
Precondition The user must have logged into the system, have team leader permissions, and users under their supervision must have submitted one overtime request which has since been approved.
Main Flow
  1. The user must head to the the overtime report tab
  2. They must then select the option to set up a new report
  3. They must then fill in all of the fields
  4. They then must select how after they want the reports to be sent to their email
Post-Condtion The team leader will then be emailed a report with summaries of the overtime reports within the timeframe specified when setting up the report
Use Case Special Claims Form
Description A user attempts to generate a special claims form
Primary actor Standard users
Secondary actor None
Precondition None
Main Flow
  1. The user must head to the the special claims tab
  2. They must then select the option to set up a new special claims form
  3. They must then fill in all of the fields
  4. They then must submit their request
Post-Condtion The user will then be able to download or get an email sent to them with the special claims form they just generated
